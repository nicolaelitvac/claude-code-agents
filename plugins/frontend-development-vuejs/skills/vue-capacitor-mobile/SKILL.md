# Vue + Capacitor Mobile

Production patterns for building Vue 3 or Nuxt apps as native mobile apps using Capacitor: project setup, native plugins, platform config, and deployment.

## When to Use This Skill

- Creating a new Vue mobile app with Capacitor
- Adding Capacitor to an existing Vue or Nuxt web app
- Integrating native device APIs (camera, geolocation, storage, etc.)
- Configuring iOS and Android builds and signing
- Handling safe areas, status bar, and splash screens
- Sharing code between web and native (single codebase)

## Core Concepts

### 1. Stack

| Layer     | Technology        |
| --------- | ----------------- |
| UI        | Vue 3 (Vite or Nuxt) |
| Bridge    | Capacitor 6+      |
| Native    | iOS (Xcode), Android (Android Studio) |

### 2. Project Layout (Vite + Vue)

```
my-app/
├── src/
│   ├── App.vue
│   ├── main.ts
│   ├── views/
│   ├── components/
│   └── composables/
├── android/          # Generated by Capacitor
├── ios/              # Generated by Capacitor
├── capacitor.config.ts
├── vite.config.ts
└── package.json
```

For Nuxt, add Capacitor to the Nuxt app root; build output is typically `.output/public` or `dist` (configure in `capacitor.config.ts`).

## Quick Start

```bash
# Create Vue app
npm create vue@latest my-app
cd my-app
npm install
npm install @capacitor/core @capacitor/cli
npx cap init "My App" "com.example.myapp"

# Add platforms
npm install @capacitor/ios @capacitor/android
npx cap add ios
npx cap add android

# Build and sync
npm run build
npx cap sync
npx cap open ios
npx cap open android
```

```typescript
// capacitor.config.ts
import type { CapacitorConfig } from '@capacitor/cli'

const config: CapacitorConfig = {
  appId: 'com.example.myapp',
  appName: 'My App',
  webDir: 'dist',
  server: {
    androidScheme: 'https',
  },
}
export default config
```

## Patterns

### Pattern 1: Detecting Native vs Web

```typescript
// composables/usePlatform.ts
import { Capacitor } from '@capacitor/core'

export function usePlatform() {
  const isNative = Capacitor.isNativePlatform()
  const platform = Capacitor.getPlatform() // 'ios' | 'android' | 'web'
  return { isNative, platform }
}
```

### Pattern 2: Camera

```typescript
// composables/useCamera.ts
import { Camera, type CameraResultType, type CameraSource } from '@capacitor/camera'

export function useCamera() {
  const takePhoto = async () => {
    const image = await Camera.getPhoto({
      quality: 90,
      allowEditing: true,
      resultType: CameraResultType.Uri,
      source: CameraSource.Camera,
    })
    return image.webPath
  }

  const pickFromGallery = async () => {
    const image = await Camera.getPhoto({
      quality: 90,
      resultType: CameraResultType.Uri,
      source: CameraSource.Photos,
    })
    return image.webPath
  }

  return { takePhoto, pickFromGallery }
}
```

### Pattern 3: Geolocation

```typescript
import { Geolocation } from '@capacitor/geolocation'

export async function getCurrentPosition() {
  const position = await Geolocation.getCurrentPosition({
    enableHighAccuracy: true,
    timeout: 10000,
  })
  return position.coords
}
```

### Pattern 4: Secure Storage (Keychain/Keystore)

```typescript
import { Preferences } from '@capacitor/preferences'

export async function setToken(token: string) {
  await Preferences.set({ key: 'authToken', value: token })
}

export async function getToken() {
  const { value } = await Preferences.get({ key: 'authToken' })
  return value
}
```

### Pattern 5: Status Bar and Safe Area

```typescript
// plugins/capacitor.client.ts (Nuxt) or in main.ts
import { StatusBar, Style } from '@capacitor/status-bar'

if (Capacitor.isNativePlatform()) {
  StatusBar.setStyle({ style: Style.Dark })
  StatusBar.setBackgroundColor({ color: '#000000' })
}
```

```css
/* Use env(safe-area-inset-*) for notches and home indicator */
.container {

  padding-bottom: env(safe-area-inset-bottom);
  padding-left: env(safe-area-inset-left);
  padding-right: env(safe-area-inset-right);
}
```

## Best Practices

### Do's

- Use **Capacitor.isNativePlatform()** before calling native plugins to avoid errors on web.
- Prefer **composables** for plugin usage so logic is reusable and testable.
- Handle **permissions** (e.g. camera, location) and user denial.
- Test on **real devices**; simulators can behave differently.
- Set **webDir** correctly in capacitor.config.ts to your build output.

### Don'ts

- Don't assume browser APIs (e.g. window.xxx) without checking on native.
- Don't ship without configuring **app icons and splash** in native projects.
- Don't forget **safe area** insets for notches and home indicator.
- Don't store secrets in Preferences without considering encryption (use @capacitor/preferences or a secure store plugin).
